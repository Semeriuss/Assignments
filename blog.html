What's New in CSS3

CSS3 is the latest version of the Cascading Style Sheets language and aims at extending CSS2.1. The following are the major additions in CSS3.

New Animative Properties

There are quite a few new properties in CSS3, many of which do what web page authors have been doing for years with various contortions of HTML, images, and JavaScript. Examples:

opacity: 0.5; Makes an element 50% opaque
border-radius: 6px; Creates a rounded effect with a 6px curvature on each corner
box-shadow: 5px 5px 10px #373737; A shadow 5px long, 5px high, a blur of 10px and a dark grey color.
text-shadow: topOffset leftOffset blurRadius color; Adds shadow to individual characters in text nodes.
background: linear-gradient (to right[direction], red[color-stop1],yellow [color-stop2], …); Displays a linear gradient of colors which is specified in the direction.
background: radial-gradient(circle [shape size at position – default is ellipse],red, yellow, green); Creates a radial color gradient.
background-image: url(firstImage.jpg), url(secondImage.gif),url(thirdImage.png); Provides us with the ability to add more than one background image to any element, even to pseudo-elements.
transition: opacity 0.5s ease-in-out; The transition property specifies a property to transition into and out of, (in this case opacity), how long to take to do the transition and the easing function, so it’s gradual.
opacity: 0; By setting opacity to 0, say on a hover event, we can create a disappear/reappear animation.

New Layouts

There are a couple of powerful new ways to lay out your page with CSS that go beyond positioning and are much easier to use. Examples:

display: table;
display: table-cell; This gives you a table layout without the HTML tables.
display: flexbox;
flex-order: 1; With flexbox you have greater control over how the browser flows boxes, like div's onto the page.

New Pseudo-classes

In CSS1 and CSS2, pseudo-elements were preceded by single colons, just as pseudo-classes were. In CSS3 and later, pseudo-elements use double colons to distinguish them from pseudo-classes. For historical reasons, browsers will support both single and double colons on pseudo-elements, but the double-colon syntax is recommended.

:root - Allows designers to point to the root element of a document. In HTML, it is html since :root is generic, it allows a designer to select the root element of an XML document without necessarily knowing it’s name.
Eg. root{overflow: auto} Permits scrollbars when needed in a document, this rule would work.
:last-child - Added as a complement to the first-child selector, which can select the last element named of a parent element.
Eg. div.article > p:last-child{font-style: italic;} Changes the font for last paragraph of a site with articles contained in div class=’article’ /div tags, where each has a last paragraph with some information that needs to be uniformly stylized.
:target - Selector draw the user’s attention to a span of text when the user clicks on a same-page link.
Eg. span.notice:target{font-size: 2em; font-style; bold;}
:not - The negation pseudo-class selector selects specified elements that doesn’t meet a particular specification.
Eg. img:not ([border]) {border: 1;} Puts a border around images that don’t have a border specified.

New Colors

New way of describing colors. Previous versions either required hexadecimal format such as #FFF, or #FFFFFF for white). It was also possible to declare colors using the rgb() notation either integers (0-255) or percentages. In CSS3 color module, the color keyword list has been extended to include 147 additional keyword colors (generally well supported).

rgba() - Works just like RGB representing red, green, and blue pixel values, except it adds a fourth value: alpha, the opacity level or alpha transparency level. For the alpha value, 1 means fully opaque, 0, is fully transparent, and 0.5 is 50% opaque. Any number between 0 and 1 is viable.
hsl() and hsla() Stands for hue, saturation, and lightness. Unlike RGB, where you need to manipulate the saturation or brightness of a color by changing all three color values in concert, with HSL you can tweak either just the saturation or the lightness while keeping the same base hue. The syntax for HSL comprises an integer value for hue, and percentage values for saturation and lightness.

    The hsl()/hsla() declaration accepts three values:

  • The hue in degrees from 0 to 359. Some examples are: 0 = red, 60 = yellow, 120 = green, 180= cyan, 240=blue, and 300 = magenta. This is based on the VIBGYOR-visible light wavelength distribution.
  • The saturation as a percentage with 100% being the norm. Saturation of 100% will be the full hue, and saturation of 0 will give you a shade of gray – essentially causing the hue value to be ignored.
  • A percentage for lightness with 50% being the norm. A lightness of 100% will be white, 50% will be the actual hue, and 0% will be black.
  • The 'a', like the rgba() stands for alpha transparency.

New Selectors

There are a whole slew of new selectors, including nth-child, which lets you target specific child elements enclosed in an element. In CSS1, matching of elements by type, class, and/or ID was allowed. CSS2 introduced several attribute selectors and added pseudo-elements, pseudo-classes, and combinators. CSS3 expanded three more attribute selectors which allow selecting substrings.

Element[attr^=val]
Eg. a[href^=’http://developers.’]{color: crimson’} Matches any element E whose attribute a starts with the value http://develops.’
Element[attr$=val]
Eg. a[href$=’.java’] {color: red;} Matches any element a whose attribute href ends in .java.
Element[attr*=val]
Eg. img[src*=’gallery’]{border-color: #a7e7ec #BBB #FFF #a7e7ec;} Matches any element img whose attribute src matches ‘gallery’, except the value can be part of a word.
ul li:nth-child(2n) { color: gray; } This means: select every alternating rows in a list item and set the background color to gray.
